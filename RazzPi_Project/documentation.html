Introduction

Nikolai the Raspberry Pi is a project utilizing many different sensors and capabilities of the Raspberry Pi to create a “sentient” machine that one can interact with in many ways. Assisted by a supplementary HTML page that explains each of Nikolai’s functions, the user of this project can tell Nikolai to perform multiple tasks using an IR Remote. This tasks range from taking a picture with a camera to lighting up LEDs with a button to doing basic math. While the user activates different functions Nikolai can perform, the Raspberry Pi will communicate with the user through an LCD Display.

This project, while giving a sample of the many different capabilities of a Raspberry Pi, lets its users experience the basics of AI and electronics. Nikolai will demonstrate how one can make their own simple “sentient robot” and introduce people to concepts that may soon become very prevalent in our society with the advancements of AI and robotics.

GPIO Wiring

This project makes use of the raspberry pi’s GPIO pins to utilize many different sensors and devices. Because of the ease of use and expandability it provides, this project uses a breadboard connected to a GPIO Extension Board to wire the devices. As this project uses many input pins for many devices, it is not possible to build this project without this extension.

The specific wiring of the project itself is found in the following list. A guide to the numbering of the pins is found in the blue Understanding the GPIO booklet found in the Viaboot Raspberry Pi kit.

  • Four jumper cables connecting power and ground to the columns of the breadboard- the simplest setup is pin 3V3 (1) to the left positive column, pin GND (9) to left negative, and then pin 5V0 (2) to right positive and pin GND (6) to right negative
  • The LCD1602 Display connects its GND and VCC wires to a negative and positive column respectively, and its SDA wire to the SDA1 pin (3) and its SCL wire to SCL1 (5)

Note: the next devices are interchangeable in which GPIO pins they use for input. However, the user must modify their device code to fit such a setup if they wish to use different pins.

  • The Ultrasonic Ranging device connects its GND and VCC wires to a negative and positive column respectively,  and its TRIG wire connects to GPIO12 (32) and its ECHO wire to GPIO24 (18)
  • The Tilt Switch connects its GND and VCC wires to a negative and positive column respectively,  and its SIG wire connects to GPIO22 (15)
  • The IR Receiver connects its GND and VCC wires to a negative and positive column respectively,  and its SIG wire connects to GPIO23 (16)
  • The Button connects its GND and VCC wires to a negative and positive column respectively,  and its SIG wire connects to GPIO20 (38)
  • The Touch Switch connects its GND and VCC wires to a negative and positive column respectively,  and its SIG wire connects to GPIO17 (11)
  • The Passive Buzzer connects its GND and VCC wires to a negative and positive column respectively,  and its SIG wire connects to GPIO16 (36)
  • The Dual-Color LED connects its GND to a negative column,  and its G wire connects to GPIO26 (37) and its R wire connects to GPIO13 (33)

In addition to the GPIO devices, the project also uses a camera connected to the Raspberry Pi. This is connected in the normal slot used for cameras on the Raspberry Pi itself. Information for setting up this device can be found on the official Raspberry Pi website. https://www.raspberrypi.org/documentation/usage/camera/python/README.md 

Lastly, there are two LED lights used in the button function of the project. Connect the negative side of these LEDS to a negative column, and then use jumper cables to connect the positive side of these to GPIO5 (29) and GPIO21 (40) each.

Usage

Despite the large number of devices utilized by Nikolai, the usage of the project is made simple by the IR Remote and helping HTML page. To get Nikolai to perform a certain task, one must press a certain number on the IR Remote. All of these functions and their corresponding numbers are found on the HTML page. In addition to the numbers, the HTML page will display a message explaining what will occur to the user after they have pressed a button. After this is finished, Nikolai will display a message on its LCD display based on what occured.

For example, if one pressed button 3 on the IR Remote, Nikolai would run its button program and wait for the user to press the button. The HTML page will prompt the user to do so once button 3 has been pressed. Once the button (sensor) is pressed, the two LEDs will light up. After doing this a few times, the program will stop, Nikolai will display a message, and the user can continue calling other functions.

Once the user is done using Nikolai, they can press the power button on the IR Remote, and Nikolai will display “Bye!” and the program will stop taking number input. Once the power button is hit again, the program will resume like normal.

Software

All software needed for building this project can be found here: https://drive.google.com/file/d/1iySOhy6Fprdbms4D71lwJxMWnyLyuqIT/view?usp=sharing 

To use the Raspberry Pi itself, the user must flash a copy of the Raspbian OS onto their MicroSD card and insert it in the Raspberry Pi.

In order for the HTML page to work, the user must install apache2 server software and cgi for python. https://courses.centre.edu/pluginfile.php/77328/mod_resource/content/1/install_apache.pdf 

Nikolai uses a number of programs to run all of its devices and its HTML page. However, the ircontrol.py program, the program that works the IR Remote device, acts as a “main” program for the project. Whenever any button is pushed on the remote, the ircontrol.py program processes this input to decide which program to call. For example, if I pressed button number 2, the ircontrol.py would process this signal, process it as option 2, send this data to the final.py program (which runs the HTML page, discussed later), and calls the touch_switch.py program. After this program has finished, the ircontrol.py program will call lcd1602.py with an argument of which option was called. From there, the lcd1602.py will display the respective result of the program based on the given number argument.

Each device program, not including the ircontrol.py, is modified to quit after a certain point is reached. This allows for the program to continue on to another function or to run the same one again. Some devices, such as the button and tilt switch, are designed to finish after they have been triggered a set number of times. Others, like the camera and ultrasonic ranging device, will wait a few seconds for the user to set up, and then perform their function once.

The HTML page, on the other hand, is fully autonomous and does not require extra user input. The HTML page will start up at the same time as the rest of the program. One can also view this on a phone or another internet capable device. To do this, the user must go to http://10.14.2.243/start.html (or the respective IP Address of a copy).

From there, they will push a start button that will display the HTML page and its contents by running final.py . This program will update the page once a device has been called, giving instructions for what will occur. This page also displays the image taken in option 8, the camera option.

In order for the program to correctly work, all programs, other than start.html, must be located in the /usr/lib/cgi-bin/ . The start.html file must in turn be located within /var/www/html/ .
